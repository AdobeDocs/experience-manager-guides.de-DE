# Einführung in die Redaktion {#id181OD0QE05Z}

Experience Manager Guides verfügt über einen benutzerfreundlichen Editor zum Erstellen und Bearbeiten strukturierter Dokumente. Der Editor verbirgt dem Autor alle Komplexitäten der DITA-Struktur. Der Editor stellt eine Liste der DITA-Elemente bereit, die ein Benutzer normalerweise in einem Dokument arbeiten muss.

## Neuer Editor

>[!NOTE]
>
> In Experience Manager Guides as a Cloud Service ist ab Version 2026.05.0 ein neuer Editor verfügbar. Wenden Sie sich an Ihr Customer Success-Team , um diese Funktion zu aktivieren.

Der neue Editor bietet eine signifikante Weiterentwicklung des Authoring-Erlebnisses mit fokussierten Verbesserungen in Bezug auf Leistung, Benutzerfreundlichkeit und Benutzeroberfläche/UX. Sie wurde speziell für DITA-Karten und DITA-Themen entwickelt und ermöglicht effizientes Authoring großer, komplexer und hochgradig strukturierter Inhalte, wobei sie vollständig abwärtskompatibel bleibt und keine Auswirkungen auf Nicht-DITA-Inhalte hat.

### Wichtigste Funktionen

- Leistungsverbesserungen für große Dateien: Verbessert die Reaktionsfähigkeit bei großen DITA-Dateien mit Unterstützung für Rückgängig-/Wiederholen- und unsaubere Markierungen, wodurch ein zuverlässiges Änderungs-Tracking ohne Leistungsbeeinträchtigung gewährleistet ist.
- Modernisierte Tabellenbearbeitung: Bietet ein intuitiveres Tabellenbearbeitungs-Erlebnis mit Drag-and-Drop-Neuanordnung, kontextuellen Symbolleisten und dem Einfügen großer Zeilen oder Spalten für eine schnellere und genauere Tabellenverwaltung.
- Verbesserter Autorenmodus: Bietet eine konfigurierbare Sichtbarkeit von XML-Kommentaren und -Attributen direkt im Autorenmodus, wodurch die Notwendigkeit verringert wird, bei der Arbeit mit strukturierten Inhalten zur Source-Ansicht zu wechseln.
- Erweiterte Bearbeitungsmodi und Tools: Führt den nebeneinander angezeigten Authoring- und Source-Modus für Themen sowie ein Schnelleinfügemenü ein, das eine schnellere strukturierte Inhaltserstellung bei minimalem Kontextwechsel ermöglicht.
- Verbesserungen der Benutzeroberfläche und der Benutzeroberfläche: Verbessert die Lesbarkeit und den visuellen Komfort durch ein dunkles Design und eine umfangreichere Standard-CSS im Autoren- und Vorschaumodus, wodurch eine modernere Bearbeitungsschnittstelle bereitgestellt wird.
- Einstellungen für den zentralen Editor: Bietet ein einheitliches Einstellungsbedienfeld, um das Authoring-Erlebnis zu personalisieren, einschließlich der Kontrolle über Tags, Attribute, XML-Kommentare, unterbrechungsfreie Leerzeichen und das Menü „Schnelleinfügung“ im Editor.

### Einschränkungen des neuen Editors

Der neue Editor unterstützt derzeit nicht die folgenden Funktionen. Diese Funktionen sollen in kommenden Versionen eingeführt werden.

- Authoring mit aktivierter Option „Änderungen verfolgen“ (alle vorhandenen verfolgten Änderungen werden beibehalten und als regulärer Inhalt behandelt)
- Neue Zitate (alle vorhandenen Zitate im Inhalt werden beibehalten)
- Bildgröße in der Autorenansicht
- Verwenden von Daten aus Datenquellen (z. B. JIRA, SQL-Datenbanken, PIM oder PLM)

>[!NOTE]
>
>Die Layout-Ansicht für DITA-Zuordnungen wird mit dem neuen Editor nicht unterstützt.


## Alter Editor

Der alte Editor in Experience Manager Guides bietet ein einfaches und zuverlässiges Authoring-Erlebnis für strukturierte Inhalte, indem er die Komplexität von DITA abstrahiert und gleichzeitig die Einhaltung unterstützter Standards wie DITA 1.2, 1.3, Lightweight DITA und spezialisiertes DITA sicherstellt. Es bietet wichtige Tools für die Arbeit mit gemeinsamen Block- und Inline-Elementen, wodurch es sich für die konsistente und kontrollierte Inhaltserstellung in einem strukturierten Framework eignet.
